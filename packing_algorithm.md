# Алгоритм упаковки молекул в кубическую ячейку

В этом файле описывается математическая постановка и алгоритм, реализованный в скриптах `pack_xyz.py` и `build_system_from_json.py`.

## 1. Постановка задачи

Дан набор типов молекул
\[
M_k,\quad k = 1,\dots,K,
\]
каждый тип задаётся:

- количеством атомов \(N_k\),
- вектором атомных символов \(\{s_{k,i}\}_{i=1}^{N_k}\),
- координатами атомов в собственной системе отсчёта
  \[
  \mathbf{r}_{k,i} \in \mathbb{R}^3,\quad i = 1,\dots,N_k,
  \]
  центрированными так, что
  \[
  \frac{1}{N_k}\sum_{i=1}^{N_k} \mathbf{r}_{k,i} = \mathbf{0}.
  \]

Для каждого типа молекулы задано целое число экземпляров \(c_k \in \mathbb{N}\).

Необходимо разместить все молекулы внутри кубической области (ячейки)
\[
Q(L) = [0, L] \times [0, L] \times [0, L]
\]
так, чтобы выполнялось минимальное межъядерное расстояние
\[
\lVert \mathbf{R}_{a} - \mathbf{R}_{b} \rVert \ge d_{\min}
\]
для любых двух атомов \(a \neq b\), где \(\mathbf{R}_a, \mathbf{R}_b\) — их глобальные координаты.

Требуется:

1. Найти допустимую конфигурацию атомов в некотором кубе \(Q(L)\).
2. При автоматическом выборе объёма — подобрать такой \(L\), который (приблизительно) минимален при сохранении выполнимости условий.

## 2. Параметризация положения молекулы

Каждый экземпляр молекулы \(M_k\) в пространстве задаётся:

- ортогональной матрицей поворота \(R \in SO(3)\), \(\det R = 1\),
- вектором сдвига (центра масс молекулы в ячейке) \(\mathbf{t} \in \mathbb{R}^3\).

Тогда глобальные координаты атома \(i\) этой молекулы:
\[
\mathbf{R}_{k,i} = R\,\mathbf{r}_{k,i} + \mathbf{t}.
\]

Условие нахождения атомов внутри куба \(Q(L)\) переписывается как
\[
0 \le (R\,\mathbf{r}_{k,i} + \mathbf{t})_j \le L,\quad
j = 1,2,3,\ i=1,\dots,N_k.
\]

В алгоритме используется более простой, но эквивалентный способ: сначала строится повёрнутая конфигурация
\[
\tilde{\mathbf{r}}_{k,i} = R\,\mathbf{r}_{k,i},
\]
затем выбирается вектор \(\mathbf{t}\) так, что
\[
\mathbf{R}_{k,i} = \tilde{\mathbf{r}}_{k,i} + \mathbf{t} \in Q(L).
\]
Практически это реализуется выбором \(\mathbf{t}\) в пределах \([0,L]^3\) при условии, что сами молекулы достаточно компактны по сравнению с \(L\).

## 3. Генерация случайного поворота

Поворот \(R \in SO(3)\) выбирается равномерно по Haar‑мере на группе \(SO(3)\) через равномерно распределённый кватернион \(q = (q_1, q_2, q_3, q_4)\), \(\lVert q \rVert = 1\).

Используется стандартный метод:

- генерируются независимые случайные величины \(u_1, u_2, u_3 \sim \mathrm{Unif}(0,1)\),
- кватернион задаётся как
  \[
  \begin{aligned}
    q_1 &= \sqrt{1-u_1}\,\sin(2\pi u_2),\\
    q_2 &= \sqrt{1-u_1}\,\cos(2\pi u_2),\\
    q_3 &= \sqrt{u_1}\,\sin(2\pi u_3),\\
    q_4 &= \sqrt{u_1}\,\cos(2\pi u_3),
  \end{aligned}
  \]
- из \(q\) строится матрица поворота \(R(q)\) по стандартным формулам.

Таким образом, ориентации молекул в пространстве распределены равномерно.

## 4. Условие минимального расстояния

Пусть уже размещены атомы с координатами \(\{\mathbf{R}_b\}_{b \in B}\) (объединение всех ранее добавленных молекул).

При попытке добавить новую молекулу получаем набор новых координат \(\{\mathbf{R}_a\}_{a \in A}\) (атомы текущей молекулы).

Требование минимального расстояния:
\[
\min_{a \in A,\ b \in B} \lVert \mathbf{R}_a - \mathbf{R}_b \rVert^2 \ge d_{\min}^2.
\]

На практике вычисляется матрица попарных квадратов расстояний
\[
D_{ab}^2 = \lVert \mathbf{R}_a - \mathbf{R}_b \rVert^2
\]
и проверяется условие
\[
\min_{a,b} D_{ab}^2 \;\ge\; d_{\min}^2.
\]

Если множество \(B\) пусто (размещается первая молекула), проверка не выполняется.

## 5. Последовательная случайная укладка (Random Sequential Adsorption)

Алгоритм упаковки реализован как **последовательная случайная укладка**:

1. Имеется список типов \((M_k, c_k)\). Формируем последовательность из всех экземпляров молекул (можно мысленно считать, что порядок произвольный, в коде — по циклам).
2. Для каждого экземпляра молекулы:
   1. Пытаемся найти допустимую позицию, делая не более \(N_{\text{trial}}\) попыток (параметр `max_attempts_per_mol`).
   2. На каждой попытке:
      - генерируется случайный поворот \(R \in SO(3)\),
      - для каждого атома вычисляются локальные координаты \(\tilde{\mathbf{r}}_{k,i} = R\,\mathbf{r}_{k,i}\),
      - выбирается случайный сдвиг \(\mathbf{t}\) с независимыми компонентами
        \[
        t_j \sim \mathrm{Unif}(0, L),\quad j=1,2,3,
        \]
      - получаем глобальные координаты \(\mathbf{R}_{k,i} = \tilde{\mathbf{r}}_{k,i} + \mathbf{t}\),
      - проверяем условие минимального расстояния до уже размещённых атомов.
   3. Если условие выполняется, текущая молекула считается размещённой, её атомы добавляются в множество \(B\).
   4. Если после \(N_{\text{trial}}\) попыток допустимое положение не найдено, алгоритм для данного размера ячейки считает упаковку невозможной и генерирует ошибку.

Последовательность добавления фиксирована, «откаты» (переразмещение ранее уложенных молекул) не выполняются — это упрощает алгоритм и делает его жадным.

## 6. Оценка минимального размера куба

При автоматическом выборе объёма (функция `pack_system_auto_box`) сначала вычисляется грубая оценка минимального размера куба на основе суммарного объёма «шаров», аппроксимирующих молекулы.

Для каждого типа молекулы \(M_k\):

1. Для её центрированных координат \(\{\mathbf{r}_{k,i}\}_{i=1}^{N_k}\) оценивается радиус
   \[
   R_k = \max_{1 \le i \le N_k} \|\mathbf{r}_{k,i}\| + \Delta,
   \]
   где \(\Delta = \tfrac{1}{2} d_{\min}\) — дополнительный зазор.
2. Молекула аппроксимируется шаром радиуса \(R_k\), её объём
   \[
   V_k = \frac{4}{3}\pi R_k^3.
   \]
3. С учётом количества экземпляров \(c_k\) суммарный объём:
   \[
   V_{\text{tot}} = \sum_{k=1}^{K} c_k\,V_k.
   \]

Предполагая желаемую «плотность упаковки» (packing fraction) \(\varphi \in (0,1)\) (в коде \(\varphi \approx 0{,}6\)), оцениваем объём куба:
\[
L^3 \approx \frac{V_{\text{tot}}}{\varphi},
\quad
L \approx \left(\frac{V_{\text{tot}}}{\varphi}\right)^{1/3}.
\]

Также задаётся минимальное значение \(L_{\min}\) (например, \(5 \,\text{Å}\)), чтобы исключить вырожденные случаи:
\[
L_0 = \max\{L, L_{\min}\}.
\]

Этот \(L_0\) используется как начальная оценка размера ячейки.

## 7. Итеративное увеличение размера ячейки

Пусть начальный размер ячейки \(L_0\). Алгоритм подбора:

1. Положить \(L = L_0\).
2. Запустить алгоритм упаковки (раздел 5) для текущего \(L\).
3. Если упаковка успешна — вернуть найденную конфигурацию и значение \(L\).
4. Если упаковка неуспешна (для данного \(L\) не удалось разместить какую‑то молекулу даже после \(N_{\text{trial}}\) попыток), увеличить размер ячейки
   \[
   L \leftarrow \alpha L,\quad \alpha > 1
   \]
   (в коде \(\alpha \approx 1{,}10\)) и повторить шаги 2–4.
5. Если \(L\) превысил заранее заданный максимум
   \[
   L_{\max} = \beta L_0,\quad \beta > 1
   \]
   (например, \(\beta = 3\)), алгоритм останавливается с ошибкой: предполагается, что при столь большом объёме и данном \(d_{\min}\) система вряд ли выполнима (или параметр \(d_{\min}\) слишком завышен).

Таким образом, итоговый размер ячейки \(\hat{L}\) определяется адаптивно:
\[
\hat{L} = \min\{L : \text{алгоритм упаковки успешен}\},
\]
где поиск ведётся по геометрической сетке \(L_0, \alpha L_0, \alpha^2 L_0, \dots\) с ограничением сверху \(L_{\max}\).

## 8. Свойства и ограничения алгоритма

- Алгоритм **жадный**: размещённые молекулы не перерасполагаются при неудаче, что упрощает реализацию, но может приводить к неидеально плотной упаковке.
- За счёт случайных поворотов и сдвигов результат **недетерминированный**: разные запуски при одинаковых параметрах могут давать разные конфигурации (но все удовлетворяют условию \(d_{\min}\)).
- Оценка минимального размера ячейки основана на сферической аппроксимации молекул и целевой плотности \(\varphi\); это **приближение**, но на практике даёт разумную стартовую точку для итеративного увеличения объёма.
- Ограничение на число попыток \(N_{\text{trial}}\) играет роль «тайм‑аута» на поиск позиции одной молекулы: при слишком плотных системах или слишком большом \(d_{\min}\) алгоритм может завершиться с ошибкой даже для достаточно большого \(L\).

Алгоритм ориентирован на быструю генерацию стартовых конфигураций для моделирования (МД, МК и т.п.), а не на строго оптимальную по плотности упаковку.
